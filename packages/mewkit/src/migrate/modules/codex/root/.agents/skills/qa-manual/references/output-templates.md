```markdown
| Flow        | Steps | Passed | Failed | Skipped | Status              |

#### {Flow name} — Step {N}: {step description}

**Expected:** {expected outcome from spec}

**Evidence:** {screenshot path or error message}

**Verdict:** {PASS (all non-skipped flows passed) | FAIL (Z flows failed)}
```

## Playwright E2E Code Structure Rules

### Rule 1: One describe per file

Every `.spec.ts` file MUST have exactly one `test.describe` block. Multiple describes → split into separate files.

### Rule 2: Feature folder with `common/`

Every feature test MUST have a `common/` folder containing reusable helpers:

```
tests/e2e/{feature-name}/
├── common/
│   ├── {featureName}Selectors.ts      # All selectors/locators for this feature
│   ├── {featureName}Assertions.ts     # Reusable assertion helpers
│   ├── {featureName}Intercepts.ts     # API intercept/mock definitions
│   └── {featureName}Helpers.ts        # (optional) Shared setup, navigation, utilities
└── {feature-name}.spec.ts             # Test file — one describe block
```

**Naming:** folder = kebab-case, common files = camelCase matching feature name.

## Playwright E2E Code Template

### `common/{featureName}Selectors.ts`

```typescript
// Generated by mk:qa-manual
// Feature: {feature name}

import type { Page } from "@playwright/test";

const selectorsCard = {
  // Form elements
  emailInput: (page: Page) => page.getByTestId("{feature}-email"),
  passwordInput: (page: Page) => page.getByTestId("{feature}-password"),
  submitButton: (page: Page) => page.getByRole("button", { name: "Submit" }),

  // Content elements
  successMessage: (page: Page) => page.getByText("{expected success text}"),
  errorMessage: (page: Page) => page.getByText("{expected error text}"),
};

export default {
  selectorsCard,
};
```

### `common/{featureName}Assertions.ts`

```typescript
// Generated by mk:qa-manual
// Feature: {feature name}

import { expect, type Page } from '@playwright/test';
import {featureName}Selectors from './{featureName}Selectors';

const shouldBeOnPage = async (page: Page, urlPattern: RegExp) => {
  await expect(page).toHaveURL(urlPattern);
};

const shouldShowSuccess = async (page: Page) => {
  await expect({featureName}Selectors.successMessage(page)).toBeVisible();
};

const shouldShowError = async (page: Page) => {
  await expect({featureName}Selectors.errorMessage(page)).toBeVisible();
};

export default {
  shouldBeOnPage,
  shouldShowSuccess,
  shouldShowError,
}
```


### `{feature-name}.spec.ts`

```typescript
// Generated by mk:qa-manual
// Source spec: {spec file path}
// Generated at: {YYYY-MM-DD HH:MM UTC}
// Feature: {feature name}

import { test } from '@playwright/test';
import {featureName}Selectors from './common/{featureName}Selectors.ts';
import {featureName}Assertions from './common/{featureName}Assertions.ts';
import {featureName}Intercepts from './common/{featureName}Intercepts.ts';

test.describe('{Feature Name}', () => {
  test.beforeEach(async ({ page }) => {
    await page.goto('{base url}');
  });

  test('{flow step 1 description}', async ({ page }) => {
    // Arrange
    await {featureName}Intercepts.mockSuccessResponse(page);

    // Act
    await {featureName}Selectors.selectorsCard.emailInput(page).fill('{value}');
    await {featureName}Selectors.selectorsCard.passwordInput(page).fill('{value}');
    await {featureName}Selectors.selectorsCard.submitButton(page).click();

    // Assert
    await {featureName}Assertions.shouldBeOnPage(page, /.*dashboard/);
    await {featureName}Assertions.shouldShowSuccess(page);
  });

  test('{flow step 2 description}', async ({ page }) => {
    // Arrange
    await {featureName}Intercepts.mockErrorResponse(page);

    // Act
    await {featureName}Selectors.selectorsCard.submitButton(page).click();

    // Assert
    await {featureName}Assertions.shouldShowError(page);
  });
});
```

### Locator Rules (MANDATORY)

| Priority | Locator                                   | When to use                                                            |
| -------- | ----------------------------------------- | ---------------------------------------------------------------------- |
| 1st      | `getByTestId('submit-btn')`               | **If codebase uses test ID attributes** — most stable, deployment-safe |
| 2nd      | `getByRole('button', { name: 'Submit' })` | Any element with ARIA role (semantic)                                  |
| 3rd      | `getByLabel('Email')`                     | Form fields with labels                                                |
| 4th      | `getByText('Welcome')`                    | Visible text content                                                   |
| 5th      | `getByPlaceholder('Search...')`           | Input placeholders                                                     |
| NEVER    | `page.locator('.css-class')`              | CSS selectors are fragile                                              |
| NEVER    | `page.locator('#id')`                     | IDs change across deployments                                          |

### Test ID Detection (run BEFORE generating code)

Codebases use different test ID attribute conventions. Detect which one this project uses:

```bash
# Step 1: Check source files for test ID attributes
echo "=== Checking test ID convention ==="
DATA_TESTID=$(grep -rl "data-testid" src/ --include="*.vue" --include="*.tsx" --include="*.jsx" --include="*.ts" 2>/dev/null | wc -l | tr -d ' ')
DATA_CY=$(grep -rl "data-cy" src/ --include="*.vue" --include="*.tsx" --include="*.jsx" --include="*.ts" 2>/dev/null | wc -l | tr -d ' ')
DATA_TEST=$(grep -rl "data-test\b" src/ --include="*.vue" --include="*.tsx" --include="*.jsx" --include="*.ts" 2>/dev/null | wc -l | tr -d ' ')
echo "data-testid: $DATA_TESTID files"
echo "data-cy: $DATA_CY files"
echo "data-test: $DATA_TEST files"

# Step 2: Check Playwright config for custom testIdAttribute
grep -r "testIdAttribute" playwright.config.* 2>/dev/null || echo "No custom testIdAttribute configured"
```

**Routing based on detection results:**

| Detection                            | `getByTestId` maps to                           | Playwright config needed                                     |
| ------------------------------------ | ----------------------------------------------- | ------------------------------------------------------------ |
| `data-testid` found (default)        | `data-testid` attribute                         | None (Playwright default)                                    |
| `data-cy` found (Cypress convention) | `data-cy` attribute                             | Add `testIdAttribute: 'data-cy'` to `playwright.config.ts`   |
| `data-test` found                    | `data-test` attribute                           | Add `testIdAttribute: 'data-test'` to `playwright.config.ts` |
| None found                           | Skip `getByTestId` — use `getByRole` as primary | None                                                         |

**If `data-cy` or `data-test` detected**, add to generated test file header:

```typescript
// Note: This project uses data-cy attributes.
// Ensure playwright.config.ts has: use: { testIdAttribute: 'data-cy' }
```

### Verifying Selectors Exist (run AFTER generating code)

Before finalizing the .spec.ts file, verify each selector actually exists in the source:

```bash
# For each getByTestId('xxx') in the generated test, verify the attribute exists in source
grep -rn "data-testid=\"submit-btn\"" src/ --include="*.vue" --include="*.tsx" --include="*.jsx" --include="*.ts" 2>/dev/null
# Or for data-cy:
grep -rn "data-cy=\"submit-btn\"" src/ --include="*.vue" --include="*.tsx" --include="*.jsx" --include="*.ts" 2>/dev/null
```

If a selector is NOT found in source:

1. **Warn** in the generated code: `// WARNING: data-testid="xxx" not found in source — may need to be added`
2. **Fall back** to `getByRole` or `getByLabel` for that element
3. **List** all missing test IDs at the end of generation for the developer to add
