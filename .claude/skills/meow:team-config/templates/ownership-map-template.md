# Ownership Map

> Generated by `meow:team-config` for parallel execution.
> Each agent owns specific file paths. Ownership violations trigger STOP.

## Agent Assignments

| Agent | Worktree | Owned Files | Task |
|-------|----------|-------------|------|
| agent-1 | `.worktrees/agent-1/` | `src/api/*`, `src/routes/*` | [task description] |
| agent-2 | `.worktrees/agent-2/` | `src/components/*`, `src/pages/*` | [task description] |
| agent-3 | `.worktrees/agent-3/` | `src/utils/*`, `src/lib/*` | [task description] |

## Shared Files (Sequential Pre-Step)

Files needed by multiple agents are handled BEFORE parallel execution starts:

| File | Action | Owner |
|------|--------|-------|
| `src/types/index.ts` | Update types first | orchestrator (sequential) |

## Ownership Rules

- **Read:** Any agent can read any file
- **Write/Edit:** Only the owning agent can modify owned files
- **Violation:** STOP immediately, report to orchestrator
- **Conflict:** If discovered mid-work, STOP both agents, escalate

## Validation

Before starting parallel work, verify:
- [ ] No glob overlaps between agents
- [ ] All files in the plan are covered by at least one agent
- [ ] Shared files handled in pre-step
- [ ] Worktrees created successfully
