# Specialist Routes

| Request | Route |
|---|---|
| Alternatives or implementation options | `mk:brainstorming` |
| Decision or design debate | `mk:party` |
| Stuck or overly complex work | `mk:problem-solving` |
| Product idea validation | `mk:office-hours` |
| Existing-plan review | `mk:plan-ceo-review` |
| Re-examine an output or verdict | `mk:elicit` |
| API or database design | `mk:api-design` or `mk:database` |
| Bug or root cause | `mk:investigate` |
| Diff or PR review | `mk:review` |
| Grade a running build | `mk:evaluate` |
| Security audit or threat model | `mk:cso` or `mk:vulnerability-scanner` |
| Command/workflow navigation | `mk:help` |
| Whole-repo discovery | `mk:scout` |
| Library/framework documentation | `mk:docs-finder` |
| Diagram, slides, visual artifact | `mk:preview --explain` |
| Requirements lacking tests | `mk:nyquist` |
