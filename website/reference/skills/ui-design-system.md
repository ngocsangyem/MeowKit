# meow:ui-design-system

Design intelligence for UI/UX: styles, palettes, accessibility (WCAG 2.1 AA), responsive rules, and quality checklists.

## What This Skill Does

`meow:ui-design-system` provides the design knowledge that the `ui-ux-designer` agent needs: style categories (minimalism, glassmorphism, brutalism, etc.), color palette selection by product type, typography pairing principles, WCAG 2.1 AA accessibility standards, responsive breakpoints, and a pre-delivery quality checklist.

## Core Capabilities

- **8 style categories** — minimalism, glassmorphism, neumorphism, brutalism, bento grid, dark mode, flat, skeuomorphism
- **Color palette direction** — by product type (SaaS, e-commerce, healthcare, finance, dev tools)
- **WCAG 2.1 AA standards** — contrast ratios (4.5:1/3:1), touch targets (44px), keyboard navigation, screen reader rules
- **Responsive breakpoints** — 320px+ → 768px+ → 1024px+ → 1440px+ (mobile-first)
- **Quality checklist** — visual, responsive, accessibility, performance, interaction checks

## When to Use This

::: tip Use meow:ui-design-system when...
- Creating or reviewing UI components
- Setting up a project's design system
- Checking WCAG accessibility compliance
- Selecting color palettes or typography
:::
