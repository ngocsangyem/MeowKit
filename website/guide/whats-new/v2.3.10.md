# v2.3.10 — Jira Ticket Intelligence + Confluence & Sprint Planning

Three new skills complete the spec-to-sprint pipeline: mk:jira gains ticket intelligence agents, mk:confluence reads and analyzes specs, and mk:planning-engine provides codebase-aware tech review and sprint planning.

All outputs are **research reports for human reading** — no automated ticket creation, assignment, or sprint modification.

## What Changed

### Jira Ticket Intelligence (mk:jira v2)

- **`/mk:jira evaluate PROJ-123`** — Qualitative complexity assessment (Simple/Medium/Complex) with Fibonacci range, inconsistency detection, historical precedent search
- **`/mk:jira estimate PROJ-123`** — Heuristic story point estimation with escalation triggers (auto-flags when human estimation is recommended)
- **`/mk:jira analyze PROJ-123`** — Full ticket context analysis with structured RCA output
- **MCP server recommendation:** [mcp-atlassian](https://github.com/sooperset/mcp-atlassian) (community, 49 tools) over official Rovo (13 tools, beta)

### Confluence Spec Analysis (NEW skill)

- **`/mk:confluence analyze PAGE-ID`** — Fetch Confluence pages as markdown, produce deep Spec Research Report
- **Gap detection** — `[MISSING]`, `[VAGUE]`, `[AMBIGUOUS]` tags for requirements quality
- **Multi-page assembly** — parent + child pages + comments assembled into single document
- **Suggested stories** — grouped requirements with complexity signals (PO assigns priority)

### Sprint Planning Engine (NEW skill)

- **`/mk:planning-engine review PROJ-123`** — Codebase-aware tech review: feasibility, affected files, dependencies, complexity signals
- **`/mk:planning-engine plan --tickets X,Y --capacity 40`** — Sprint planning: dependency map, grouping, sequencing, capacity analysis
- **Deterministic scripts** — `dep-graph.py` (cycle detection), `capacity-bin.py` (bin-packing)
- **Sprint goal candidate** — AI suggests draft, team negotiates the real goal
- **Scrum-aligned** — no auto-assignment, no auto-estimation, no auto-sprint-commit

### SKILL.md Restructured

mk:jira is now a **thin routing layer** (~165 lines) with MCP translation notes documenting exact tool mappings (transition IDs, assignee formats, field JSON nesting).

### Security Improvements

- **Injection defense:** All external content wrapped in `===DATA_START===` / `===DATA_END===` boundary markers
- **Tier 2 batch confirmation:** Batch creates (3+ items) require preview + confirmation
- **Rule of Two:** Both new skills are 1/3 (untrusted input only, no state changes) — safer than mk:jira

### End-to-End Pipeline

```
/mk:confluence analyze PAGE-ID  → Spec Research Report
  ↓ human reads, decides
/mk:jira create (manual)        → tickets created
/mk:jira evaluate + estimate    → complexity + points
  ↓ human reads, decides
/mk:planning-engine review      → Tech Review Report
/mk:planning-engine plan        → Planning Report
  ↓ human reads, team decides sprint
```

Every step produces a report. Human gates at every transition.

## Breaking Changes

None. All changes are additive. Existing commands work unchanged.

## Migration

No migration required. For the new skills, install [mcp-atlassian](https://github.com/sooperset/mcp-atlassian) with Confluence access:

```bash
claude mcp add -e JIRA_URL=https://your-company.atlassian.net \
  -e JIRA_USERNAME=your-email@company.com \
  -e JIRA_API_TOKEN=your-api-token \
  atlassian -- uvx mcp-atlassian
```
